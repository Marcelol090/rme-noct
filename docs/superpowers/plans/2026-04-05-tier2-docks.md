# Implementation Plan: Tier 2 Docks & Dialogs (Noct UI Parity)
**Date:** 2026-04-05
**Feature:** Tier 2 UI components (Map Properties, Waypoints, Find Item, Layers Toolbar)
**Context:** The codebase uses PyQt6 with a custom Noct Map Editor design system (Glassmorphism). C++ parity is mandatory.

## Chunk 1: Map Properties Dialog
**Objective:** Add the Map Properties modal dialog exactly as it exists in legacy C++.
**Files to Touch:**
- `pyrme/ui/dialogs/map_properties.py` (NEW)
- `pyrme/ui/main_window.py` (MODIFY: Hook to top menu `Map > Properties`)

**Code Details:**
1. Create `MapPropertiesDialog(GlassPanel, QDialog)` leveraging `GlassPanel` for theme.
2. Form layout:
  - `Description`: QTextEdit
  - `OTBM Version`: QComboBox
  - `Client Version`: QComboBox
  - `Width`, `Height`: QSpinBox (max 65000)
  - `Housefile`, `Spawnfile`, `Waypointfile`: QLineEdit
3. Verification: Ensure it sizes correctly and elements don't overflow. Run `pytest` and load shell.

## Chunk 2: Waypoints Dock
**Objective:** Add Waypoints list dock floating on the right/left side.
**Files to Touch:**
- `pyrme/ui/docks/waypoints.py` (NEW)
- `pyrme/ui/main_window.py` (MODIFY: Register dock in `setup_docks`)

**Code Details:**
1. Create `WaypointsDock(GlassDockWidget)`.
2. Inside dock: `QVBoxLayout`, leading with a `QListWidget`.
3. Bottom bar with `QHBoxLayout`: 'Add' and 'Remove' styled buttons (`AppButton` or generic themed QPushButton).
4. Inline editing behavior built-in via `QListWidgetItem` edit flags.
5. Verification: Check dock docking, moving, glass look. Run `pytest`.

## Chunk 3: Find Item Dialog
**Objective:** Add the dense Find Item dialog (Grid + List support).
**Files to Touch:**
- `pyrme/ui/dialogs/find_item.py` (NEW)
- `pyrme/ui/main_window.py` (MODIFY: Hook to menu `Edit > Find Item`)

**Code Details:**
1. Modal `QDialog`. Split layout: `QSplitter` or `QHBoxLayout`.
2. Left: Search bar, Type checkboxes (Items, Creatures, Spawns), Property checkboxes (Impassable, Immovable, Container).
3. Right: Results using `QListWidget` supporting `IconMode` and `ListMode` via a toggle button.
4. Verification: Run shell to ensure search layout resembles C++ legacy `find_item_window.cpp`. Ensure filters don't misalign.

## Chunk 4: Layers Toolbar UI Elements
**Objective:** Replace the need for a layers dock via a dedicated toolbar.
**Files to Touch:**
- `pyrme/ui/main_window.py` (MODIFY)

**Code Details:**
1. Add a `QToolBar("Layers Toolbar")` to the `MainWindow`.

4. Verification: Ensure toolbar docks beautifully with the Noct theme styling.

## Verification & Handoff
After each Chunk is executed, we invoke the `plan-document-reviewer` subagent or perform a TDD dry-run. After all chunks are complete, we review visual exactness with Stitch design intents.
